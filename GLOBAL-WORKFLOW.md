# Global Workflow - Using jscad-ai-studio from Any Directory

This document describes how to use `jscad-work` to develop JSCAD models from any project directory with Claude Code integration.

## Installation

Install the global command:

```bash
cd /home/john/src/jscad-ai-studio
npm link
```

This creates a global `jscad-work` command available anywhere.

## Workflow

### 1. Create or Navigate to Your Project Directory

```bash
cd ~/my-project
# or
mkdir ~/new-project && cd ~/new-project
```

### 2. Create a Model File

```bash
cat > my-model.js <<'EOF'
const jf = require('@jbroll/jscad-fluent');

const main = (p) => {
  p._type = 'My Model';
  p.size = { type: 'slider', default: 10, min: 5, max: 20, step: 1, label: 'Size', live: true };

  return jf.cube({ size: p.size }).colorize([0.3, 0.6, 0.8]);
};

module.exports = { main };
EOF
```

### 3. Initialize JSCAD Workspace

```bash
jscad-work my-model.js
```

This command:
- Creates a symlink from jscadui viewer to your current directory
- Generates `CLAUDE.md` with workflow instructions
- Creates `.jscad-studio` config file
- Shows you the viewer URL

### 4. Start the Viewer (if not running)

```bash
jscad-work start
```

Or manually:
```bash
cd /home/john/src/jscadui/apps/jscad-web
npm start
```

### 5. Open in Browser

```
http://127.0.0.1:5120#my-project/my-model.js
```

### 6. Start NEW Claude Code Session

**IMPORTANT**: You must start a **new** Claude Code session to pick up the CLAUDE.md file:

```bash
cd ~/my-project
claude-code
```

The new session will automatically read:
- `CLAUDE.md` (workflow instructions generated by jscad-work)
- `.jscad-studio` (configuration)
- Your model files

You **cannot** message an existing Claude Code session - each session is isolated.

## How Claude Code Priming Works

When you start a new Claude Code session in a directory, it:

1. **Reads CLAUDE.md** - Project-specific instructions
2. **Reads .claude/settings.json** - Local settings (if present)
3. **Has access to all files** - Can read your models and the jscad-fluent API docs

The `jscad-work` command generates a CLAUDE.md that:
- References the jscad-fluent API at `/home/john/src/jscad-ai-studio/llm.txt`
- Explains the parametric model format
- Shows the viewer URL for this directory
- Lists available models
- Includes code style guidelines

## Multi-Project Workflow

You can work on multiple projects simultaneously:

```bash
# Terminal 1
cd ~/project-a
jscad-work model-a.js
claude-code

# Terminal 2
cd ~/project-b
jscad-work model-b.js
claude-code

# Both projects share the same viewer
# Access via:
#   http://127.0.0.1:5120#project-a/model-a.js
#   http://127.0.0.1:5120#project-b/model-b.js
```

## Example Session

```bash
# Create new project
mkdir ~/gear-designs && cd ~/gear-designs

# Create initial model
cat > spur-gear.js <<'EOF'
const jf = require('@jbroll/jscad-fluent');
const main = (p) => {
  p._type = 'Spur Gear';
  p.teeth = { type: 'int', default: 20, min: 8, max: 60, label: 'Teeth', live: true };
  p.module = { type: 'slider', default: 2, min: 1, max: 5, step: 0.1, label: 'Module', live: true };

  // Simplified gear (real implementation would be more complex)
  const radius = p.module * p.teeth / 2;
  return jf.cylinder({ radius, height: 10 }).colorize([0.8, 0.6, 0.2]);
};
module.exports = { main };
EOF

# Initialize workspace
jscad-work spur-gear.js

# Start viewer
jscad-work start &

# Open browser to: http://127.0.0.1:5120#gear-designs/spur-gear.js

# Start Claude Code in NEW session
claude-code
# Then tell Claude: "Help me design a proper involute spur gear"
```

## Commands

| Command | Description |
|---------|-------------|
| `jscad-work` | Initialize current directory (uses first .js file found) |
| `jscad-work <model.js>` | Initialize and work on specific model |
| `jscad-work start` | Start the jscadui viewer |

## File Structure

After running `jscad-work`, your directory will have:

```
my-project/
├── my-model.js           # Your JSCAD model
├── CLAUDE.md             # Auto-generated workflow instructions for Claude
├── .jscad-studio         # Config (gitignored)
└── other-files...
```

## Tips

1. **Add to .gitignore**: Add `.jscad-studio` to your project's .gitignore
2. **Keep CLAUDE.md**: Commit CLAUDE.md so other Claude Code sessions know the workflow
3. **One viewer, many projects**: You only need one viewer running for all projects
4. **Fresh Claude sessions**: Each new project directory needs its own Claude Code session
5. **Parameter format**: Use compact one-line parameter declarations (see CLAUDE.md)

## Troubleshooting

**"No .js files found"**
- Create a model file first, then run `jscad-work`

**"Model not loading in browser"**
- Check the viewer is running: `jscad-work start`
- Check the symlink exists: `ls -la /home/john/src/jscadui/apps/jscad-web/build_dev/`
- Refresh browser with correct URL

**"Claude doesn't know the workflow"**
- Make sure you started a NEW Claude Code session in the project directory
- Check CLAUDE.md exists: `cat CLAUDE.md`
- Claude Code reads CLAUDE.md on startup, not during a session

**"How do I switch between projects?"**
- You can't in the same Claude session
- Start a new Claude Code session in the other directory
- Or tell Claude to read the other project's CLAUDE.md explicitly
